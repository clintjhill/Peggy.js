Peggy

Peggy is a Parsing Expression Grammar (PEG) generator implemented in JavaScript that is designed to be small and easy-to-use. There are other PEG JavaScript implementations out there but they are large or complicated. Peggy aims to be the better choice by remaining small and simple.

Start with some examples

Each example demonstrates the ability to create a grammar through the Peggy JavaScript API. They are progressively built, so starting with the "Additive Easy" example you will see a very small and naive implementation and build on that for "Additive Enhanced".

Peggy JavaScript API

To view the API documentation go to the JavaScript API, or read on to begin creating a grammar with the Peggy constructor.

var sample = new Peggy('additive');

Now you can build rules from that grammar with the following API.

sample.rule(name, declaration, extension);

To create the top of the parsing tree (the initial starting expression) use the root function.

sample.root(name, declaration, extension);

The name parameter provides a way for the grammar to identify the rule during processing and as well for aliasing in other rules later. The declaration parameter can be a string, regular expression or an alias to another rule in the grammar. Aliases take the form of the name preceded by a colon such as ':number'. The extension parameter is a function used to evaluate the value of the matched string. It receives a value parameter that is an object (or array) populated with matches for the given rule.

sample.root('additive', 
  sample.sequence(':number', ':plus', ':number'), 
  function(value){
    return value.number[0] + value.number[1];
});

sample.rule('number', /\d+/, function(value){ 
    return new Number(value); 
});

sample.rule('number', '+');

With rules set against the grammar you can now parse strings.

var result = sample.parse('6+6');   
// result is 12

Peggy Syntax - BETA

To create a grammar named 'demo' you could write the following Peggy syntax. The following is Beta and not currently included in the build. It is incomplete and buggy.

demo = {
    additive: (:number, :plus, :number) {{
        return value.number[0] + value.number[1];
    }},
    number: (:digit, :space) {{
        return value.digit;
    }},
    plus: ('+', :space),
    digit: (/\d+/, :space) {{
        return new Number(value[0]);
    }},
    space: /\s+/
}

Breakdown of Peggy Syntax

Using the demo grammar above the following are definitions of the components of Peggy syntax.

Grammar
demo - the name of this set of rules.
Rule
additive - a single rule name.
Non-Terminal
(:number, :plus, :number) - a Non-Terminal definition of a rule. In this case a Sequence. Repeat and Choice Non-Terminals are also available.
Terminal
/\d+/ - a Terminal definition, simply a singular term from the content of the parsing. Can be a string or a regular expression.
Alias
:number - a name to apply to another rule within the grammar.
Extension
{{ return new Number(value[0]); }} - a JavaScript block of code (wrapped with 2 curly braces) to execute that is provided a Value object.
Value
value[0]- an object that is the result of the rule evaluation.

What is Peggy and what's a PEG?

A Parsing Expression Grammar (PEG) is used to create a set of rules with which you can recognize values from within strings. They are best applied to strings that represent known sets of values in either linear or heirarchal form. Peggy is a JavaScript API to help generate a PEG.

Developers

If you would like to help out or extend Peggy - please do! Below is information about building this project.

Building Peggy.js

  • make : creates both a peggy.js and a peggy-min.js file inside of a /dist directory (with version number).
  • make lint : tests the source of Peggy.js against JSHint
  • make spec : run tests in /specs directory
  • make clean : deletes the /dist directory

Requirements to build and run tests

  • GNU Make
  • Node.js 0.5.0 or greater
  • Node.js Package Manager 1.0.0 or greater
  • jasmine-node npm install -g jasmine-node
