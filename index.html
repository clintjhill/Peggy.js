Peggy

Peggy is a Parsing Expression Grammar (PEG) generator implemented in JavaScript that is designed to be small and easy-to-use. There are other PEG JavaScript implementations out there but they are large or complicated. Peggy aims to be the better choice by remaining small and simple.

A Parsing Expression Grammar (PEG) is used to create a set of rules with which you can recognize values from within strings. They are best applied to strings that represent known sets of values in either linear or heirarchal form. Peggy is a work-in-progress but is designed to follow the work described by Brian Ford.

Peggy JavaScript API

To create a grammar begin with the Peggy constructor.

var sample = new Peggy('additive');

Now you can build rules from that grammar with the following API.

sample.rule(name, declaration, extension);

To create the top of the parsing tree (the initial starting expression) use the root function.

sample.root(name, declaration, extension);

The declaration parameter of root and rule functions can be a string, regular expression or an alias to another rule in the grammar. Aliases take the form of the name preceded by a colon: ':number'. The extension parameter is a function used to evaluate the value of the matched string of the rule. It receives a value parameter that is an object (or array) populated with matches for the given rule.

sample.root('additive', 
  sample.sequence(':number', ':plus', ':number'), 
  function(value){
    return value.number[0] + value.number[1];
});

sample.rule('number', /\d+/, function(value){ 
    return new Number(value); 
});

sample.rule('number', '+');

With rules set against the grammar you can now parse strings.

var result = sample.parse('6+6');   
// result is 12

Peggy Syntax - BETA

To create a grammar named 'demo' you could write the following Peggy syntax. The following is Beta and not ready for production use. It is incomplete and buggy.

demo = {
    additive: (:number, :plus, :number) {{
        return value.number[0] + value.number[1];
    }},
    number: (:digit, :space) {{
        return value.digit;
    }},
    plus: ('+', :space),
    digit: (/\d+/, :space) {{
        return new Number(value[0]);
    }},
    space: /\s+/
}

Breakdown of Peggy Syntax

Using the demo grammar above the following are definitions of the components of Peggy syntax.

Grammar: demo The name of this set of rules.

Rule: additive A single rule name.

Non-Terminal: (:number, :plus, :number) A Non-Terminal definition of a rule. In this case a Sequence. Repeat and Choice Non-Terminals are also available.

Terminal: /\d+/ A Terminal definition, simply a singular term from the content of the parsing. Can be a string or a regular expression.

Alias: :number A name to apply to another rule within the grammar.

Extension: {{ return new Number(value[0]); }} A JavaScript block of code (wrapped with 2 curly braces) to execute that is provided a Value object.

Value: value[0] A Value object that is the result of the expression evaluation.

Developers

If you would like to help out or extend Peggy - please do! Below is information about building this project.

Building Peggy.js

  • make : creates both a peggy.js and a peggy-min.js file inside of a /dist directory
  • make lint : tests the source of Peggy.js against JSLint
  • make spec : run tests in /specs directory
  • make clean : deletes the /dist directory

Requirements to build and run tests

  • GNU Make
  • Node.js 0.5.0 or greater
  • Node.js Package Manager 1.0.0 or greater
  • jasmine-node npm install -g jasmine-node
