Example: Additive - Easy

Creating a simple grammar with a single rule for parsing is easy:

var simple = new Pegg('simple');
simple.rule('easy', /\d+/);
simple.parse("w"); // returns Failed to parse "w"
simple.parse("9"); // returns 9

But this isn't anymore helpful than a normal Regular Expression. In fact it would be far easier to just use a Regular Expression. Peggy provides a way to add more structure to parsing strings and make complicated rules easier to use.

The following is an example of creating a grammar that will parse out a very simple addition calculation. It is simple in that it does not respect whitespace and does not account for floating-point numbers.

Grammar

The first step is to create the grammar. The next step is to provide the grammar with a root rule that will act as the entry-point for the parsing.

var sample = new Peggy('sample');

sample.root('additive', sample.all(/\d+/, '+', /\d+/), function(value){
	return new Number(value[0]) + new Number(value[2]);
});

The rule 'additive' above simply provides a sequence of terminal rules which are un-named. This is different than just providing a single rule because you are allowing for this rule to be a structure of any number of rules. The first and last parameters are regular expressions (/\d+/) that will look for a numeral and the second parameter is going to look for a '+' symbol. The function is used as an extension to the rule and will receive the value of the parse as an argument. Because the rules provided in sample.all are un-named, the value object is an array that holds the results in indexed order.
